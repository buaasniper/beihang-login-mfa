# 研究内容与创新蓝图

## 产品定位

北航登录 MFA 面向登录认证场景，目标是构建一个统一的智能风险平台，而不是单一的 anti-bot 检测工具。

系统核心逻辑是：

1. 共享底座统一接入与标准化
2. 先做 Anti-Bot
3. 再做 Device-ID / RBA
4. Offline intelligence 持续增强 Online
5. Research / Evaluation / Governance 支撑长期演进

---

## 核心创新方向

### 1. 分阶段风险判定
- 先 `Anti-Bot`
- 后 `Device-ID / RBA`
- 避免 bot 流量污染身份与风险推理

### 2. 可升级的 Device-ID
- `device-id v1`：deterministic hash-linking
- `device-id v2`：fuzzy / graph linking
- 通过统一接口实现升级与 fallback

### 3. Offline-to-Online Intelligence Loop
- Offline 不是只出报表
- Offline 会持续生成：
  - 新 rule
  - 新 threshold
  - 新 profile
  - 新 model intelligence
- 再动态更新 Online

### 4. Research Validation Layer
- research 不直接扫生产逻辑
- 研究集主要从 offline decision 派生
- 方便 rule / model / linking 算法做统一验证

### 5. Agent / Control Layer
- 双更新控制核心
- 一侧更新 online execution
- 一侧更新 offline detect / eval / research

---

## 核心模块说明

### Shared Foundation

**包含**
- Ingest API
- Normalize + Hash Engine
- hash libraries
- `bfp_event`
- pipeline state

**输入**
- Frontend SDK JSON payload

**输出**
- 统一事件事实层
- 增量调度状态

**为什么存在**
- 为 Online / Offline / Research 提供统一入口

**特点**
- 统一口径
- 可扩展
- 可增量运行

---

### Online Anti-Bot

**输入**
- `bfp_event`
- `online_rule_config`
- `risk_profiles`

**输出**
- `online_bot_result`
- bot / challenge / pass

**为什么存在**
- 作为第一阶段门禁，先筛掉明显坏流量

**特点**
- 低延迟
- 高并发
- 可解释

**创新点**
- Anti-Bot First，保护后续 identity / RBA 逻辑

---

### Device-ID / RBA

**输入**
- anti-bot pass 事件
- 设备身份表
- 风险画像

**输出**
- `online_rba_result`
- 最终登录风险判断

**为什么存在**
- 登录场景不仅要拦 bot，还要识别设备与账号风险

**特点**
- 采用统一 Device Identity Interface
- 支持 v1 / v2 双版本演进

**创新点**
- 将 deterministic identity 升级为 future fuzzy / graph identity

---

### Offline Intelligence

**输入**
- `bfp_event`
- pipeline state
- online / offline 结果资产

**输出**
- offline feature tables
- `offline_bot_result`
- `offline_rba_result`
- `risk_link`
- `risk_entity_score`

**为什么存在**
- Online 保持轻量与实时
- 更重的 intelligence 放在 Offline

**特点**
- 多时间窗口
- 可做更复杂的 linking / graph / reasoning

**创新点**
- Offline 不只是分析层，而是智能能力生产层

---

### Research Platform

**输入**
- `research_validation_dataset`
- `offline_bot_result`
- `offline_rba_result`

**输出**
- 候选规则
- 新阈值
- 模型结果
- 实验结论

**为什么存在**
- 让新规则、新模型有统一验证空间

**特点**
- 可复现
- 可比较
- 可追踪

**创新点**
- 支撑后续论文、benchmark、模型迭代

---

### Agent / Control Layer

**输入**
- research platform 输出
- offline anti-bot / offline RBA intelligence

**输出**
- 更新 `online_rule_config`
- 更新 `risk_profiles`
- 更新 offline detect / eval / research 流程

**为什么存在**
- intelligence 更新必须有统一控制中枢

**特点**
- 双更新
- 统一控制
- 支持动态演进

**创新点**
- 这是整个系统最关键的 agent-driven intelligence controller

---

## 分阶段路线

### P1
- Shared Foundation
- Online Anti-Bot
- device-id v1
- basic Online RBA
- Offline 5m / 1h 特征准备
- Offline detect
- Dashboard

### P2
- device-id v2
- identity linking / graph
- offline RBA intelligence

### P3
- research validation dataset
- research platform
- agent / control layer
- online dynamic update

### P4
- shadow evaluation
- rule lifecycle
- feature governance
- data quality
- evaluation framework
